- Пользовательский сценарий 1.1 Регистрация пользователя
-- Тестовый сценарий 1.1.1 Регистрация нового пользователя

Описание: данный сценарий проверяет возможность регистрации нового пользователя, логин/пароль которого соответствуют
требованиям. Адрес нового пользователя отсутсвует в базе данных существующих пользователей.

Предусловия: тест должен выполняться в браузере Google Chrome v.90 или выше.
Имеется новый пользователь (логин/пароль, соответсвуют требованиям), адрес электронной почты которого ещё не был
зарегистрирован.

Шаги:

1. Открыть страницу http://selenium1py.pythonanywhere.com/ru/accounts/login/
- Открыта страница с двумя формами ввода: "Войти" и "Зарегистрироваться".
Элементы и поля страницы корректно отображаются.
Поля для ввода данных открыты для редактирования.
Кнопки "Войти" и "Зарегистрироваться" корректно отображаются.

2. Ввести логин в виде адреса электронной почте в поле "Адрес электронной почты" на форме "Зарегистрироваться"
- Адрес электронной почты успешно введён.

3. Ввести пароль в поле "Пароль" на форме "Зарегистрироваться"
- Пароль успешно введён

4. Ввести такой же пароль а поле "Повторите пароль" на форме "Зарегистрироваться"
- Пароль успешно введён

5. Открыть инструменты разработчика в браузере. Нажать на кнопку "Зарегистрироваться"
- В инструментах разработчика в браузере во вкладке Network со страницы был отправлен POST-запрос и получен ответ.
Открылась страница http://selenium1py.pythonanywhere.com/ru/ с сообщением "Спасибо за регистрацию!"
Пользователь зарегистрирован и залогинен. Вверху страницы справа появились ссылки "Аккаунт" и "Выход".
